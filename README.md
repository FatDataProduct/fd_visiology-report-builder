# Visiology Report Builder

**Конструктор шаблонов регламентных отчётов для платформы Visiology**

Веб-приложение с визуальным редактором Excel-шаблонов: расставляйте блоки, вставляйте маркеры данных, добавляйте формулы и условное форматирование — затем экспортируйте готовый `.xlsx`-файл, который система Visiology заполнит реальными данными.

## Быстрый старт

```bash
# 1. Клонировать репозиторий
git clone <url>
cd fd_visiology-report-builder

# 2. Установить зависимости
npm install

# 3. Запустить dev-сервер
npm run dev
```

Откройте [http://localhost:5173](http://localhost:5173) в браузере.

Для сборки production-версии:

```bash
npm run build
```

Статические файлы появятся в папке `dist/`.

## Что такое маркеры Visiology

Маркер — это специальная метка-заполнитель вида `{{VIS_VAL_01}}`, которую вы ставите в ячейку шаблона. Когда Visiology генерирует отчёт, она находит все маркеры в `.xlsx`-файле и заменяет их реальными данными из виджетов дашборда.

Формат маркера:

```
{{VIS_VAL_01}}   — одно значение (ячейка)
{{VIS_TBL_01}}   — целая таблица
{{VIS_COL_01}}   — один столбец таблицы
```

Приложение автоматически нумерует маркеры при добавлении блоков, чтобы имена не повторялись.

## Интерфейс приложения

```
┌────────────────────────────────────────────────────────────────┐
│  HEADER: логотип · Шаблоны · Импорт · Экспорт                 │
├──────────────────┬─────────────────────────────────────────────┤
│  ЛЕВАЯ ПАНЕЛЬ    │  РАБОЧАЯ ОБЛАСТЬ (превью шаблона)           │
│                  │                                             │
│  • Блоки         │  [ Лист 1 ] [ Лист 2 ] [ + ]               │
│  • Структура     │                                             │
│  • Маркеры       │  ┌─────────────────────────────────────┐   │
│  • Печать A4     │  │  РЕГЛАМЕНТНЫЙ ОТЧЁТ                 │   │
│                  │  │  Дата: [◎VAL_01]  Период: [◎VAL_02]│   │
│                  │  │  Выручка │ [◎VAL_03] │ [◎VAL_04]   │   │
│                  │  │  [▦TBL_01 — таблица из 5 строк]    │   │
│                  │  └─────────────────────────────────────┘   │
└──────────────────┴─────────────────────────────────────────────┘
```

### Панель блоков (левая колонка)

Содержит три раздела:

**БЛОКИ** — палитра готовых блоков для перетаскивания или клика:

| Иконка | Блок | Описание |
|--------|------|----------|
| ◈ | Шапка отчёта | Заголовок, дата, отчётный период |
| ◉ | KPI-панель | 4 ключевых показателя в ряд |
| ▦ | Таблица данных | Маркер-таблица на несколько строк |
| ▥ | Столбец данных | Маркер одного столбца |
| Σ | Итоговая строка | Строка с формулами SUM |
| ─ | Разделитель | Пустая строка между секциями |
| ◻ | Подвал | Строки подписей «Подготовил / Утвердил» |

Чтобы добавить блок на лист — **кликните** по нему или **перетащите** в рабочую область.

**СТРУКТУРА** — список блоков текущего листа. Для каждого блока доступны:
- `+r` — добавить строку в блок
- `+c` — добавить столбец в блок
- `✕` — удалить блок
- Перетащите блок в списке, чтобы **изменить его порядок**

### Рабочая область (правая часть)

Показывает живой превью шаблона в виде таблицы. Обозначения ячеек:

| Вид ячейки | Значение |
|------------|----------|
| Жёлтый фон, пунктирная граница | Ячейка-маркер Visiology |
| Фиолетовый текст, пунктирная граница | Ячейка с Excel-формулой |
| Маленький ромб `◆` в углу | Настроено условное форматирование |

**Кликните по любой ячейке**, чтобы открыть редактор.

Каждая строка имеет кнопку `✕` справа для удаления.

Пунктирная горизонтальная линия показывает **границу страницы A4** (≈ 45 строк).

### Панель маркеров

В нижней части левой колонки отображается список **всех маркеров** по всем листам: имя маркера, тип (VAL / TBL / COL) и наличие дополнительных настроек (`⚙`).

### Счётчик страниц A4

Показывает:
- Общее количество строк в текущем листе
- Расчётное число страниц A4
- Прогресс-бар (зелёный — умещается в одну страницу, жёлтый — больше)

## Типы блоков

### Шапка отчёта `◈`

```
┌──────────────────────────────────┐
│       РЕГЛАМЕНТНЫЙ ОТЧЁТ        │  ← жирный, 16pt, тёмно-синий фон
│                                  │
│ Дата формирования: [◎ДАТА]  Период: [◎ПЕРИОД] │
└──────────────────────────────────┘
```
Содержит маркеры даты и отчётного периода.

### KPI-панель `◉`

```
┌──────────┬──────────┬──────────┬──────────┐
│Показ. 1  │Показ. 2  │Показ. 3  │Показ. 4  │
│ [◎VAL]   │ [◎VAL]   │ [◎VAL]   │ [◎VAL]   │
└──────────┴──────────┴──────────┴──────────┘
```
4 маркера ячеек рядом — подходит для главных метрик.

### Таблица данных `▦`

```
┌──────────┬──────────┬──────────┬──────────┐
│Колонка A │Колонка B │Колонка C │Колонка D │
│                                           │
│       [▦VIS_TBL_XX — 4 строки]           │
│                                           │
└──────────┴──────────┴──────────┴──────────┘
```
Маркер типа «Таблица» занимает объединённую область. Visiology вставит сюда целый датасет, раздвигая строки.

### Столбец данных `▥`

```
┌──────────┬──────────┬──────────┬──────────┐
│Категория │[▥COL_XX] │          │          │
│Строка 1  │          │          │          │
│Строка 2  │          │          │          │
└──────────┴──────────┴──────────┴──────────┘
```
Маркер типа «Столбец» — Visiology подставит значения из одного столбца виджета.

### Итоговая строка `Σ`

```
┌──────────┬──────────────┬──────────────┬──────────────┐
│  ИТОГО:  │=SUM(B2:B10)  │=SUM(C2:C10)  │=SUM(D2:D10)  │
└──────────┴──────────────┴──────────────┴──────────────┘
```
Ячейки с формулами — вычисляются непосредственно в Excel после открытия файла.

### Разделитель `─`

Одна пустая строка на всю ширину. Используется для визуального отступа между секциями.

### Подвал `◻`

```
┌──────────────────────────┬──────────────────────────┐
│ Подготовил: ____________ │ Утвердил: ______________ │
│ Дата: «__» _______ 20__ │ Дата: «__» _______ 20__ │
└──────────────────────────┴──────────────────────────┘
```
Строки для подписей. Легко адаптируется под любую форму отчётности.

## Типы маркеров

В редакторе ячейки раздел **«МАРКЕР VISIOLOGY»** позволяет выбрать один из трёх типов:

### ◎ Ячейка (`VAL`)

Подставляет одно значение из виджета.

Дополнительные настройки:
- **Индекс строки** — из какой строки виджета взять значение (0 = первая)
- **Индекс столбца** — из какого столбца виджета взять значение (0 = первый)

Имя маркера: `{{VIS_VAL_01}}`, `{{VIS_VAL_02}}`, ...

### ▦ Таблица (`TBL`)

Подставляет целый датасет. Visiology развернёт таблицу начиная с позиции маркера.

Дополнительные настройки:
- **Отображать заголовки столбцов** — включить / выключить строку с названиями колонок
- **Смещать строки ниже (динамическая высота)** — при включении Visiology сдвигает вниз строки, находящиеся ниже маркера, освобождая место под данные

Имя маркера: `{{VIS_TBL_01}}`, `{{VIS_TBL_02}}`, ...

### ▥ Столбец (`COL`)

Подставляет значения из одного столбца виджета.

Дополнительные настройки:
- **Индекс столбца данных** — номер столбца в виджете (0 = первый)

Имя маркера: `{{VIS_COL_01}}`, `{{VIS_COL_02}}`, ...

## Редактор ячейки

Кликните по любой ячейке в рабочей области — откроется всплывающий редактор.

| Поле | Описание |
|------|----------|
| **Значение** | Текст, маркер `{{VIS_...}}`, или Excel-формула `=SUM(...)` |
| **Формула Excel** | Флажок — пометить ячейку как формульную |
| **Фон** | Цвет фона (пикер + HEX-поле) |
| **Цвет текста** | Цвет шрифта (пикер + HEX-поле) |
| **Размер** | Размер шрифта (8–36pt) |
| **Выравнивание** | Лево / Центр / Право |
| **Жирный** | Включить жирное начертание |
| **Маркер Visiology** | Флажок + выбор типа маркера + дополнительные параметры |
| **Условное форматирование** | Правила автоматической смены цвета |

Кнопки: **Сохранить** применяет изменения, **Отмена** закрывает без сохранения.

## Условное форматирование

В редакторе ячейки нажмите **«+ правило»** в разделе «УСЛОВНОЕ ФОРМАТИРОВАНИЕ».

Каждое правило состоит из:
- **Оператор сравнения** — одно из:

| Символ | Значение |
|--------|----------|
| `<` | меньше |
| `>` | больше |
| `=` | равно |
| `≤` | не больше |
| `≥` | не меньше |
| `≠` | не равно |
| `содержит` | вхождение подстроки |

- **Порог** — значение для сравнения (число или текст)
- **Цвет фона** при срабатывании правила
- **Цвет текста** при срабатывании правила

Можно добавить несколько правил на одну ячейку. Ячейки с активными правилами помечены символом `◆` в углу.

## Excel-формулы

Любая ячейка может содержать Excel-формулу. Чтобы её активировать:

1. Откройте редактор ячейки
2. Введите формулу в поле «Значение», начиная с `=`
3. Поставьте галочку **«Формула Excel»**

Примеры:

```
=SUM(B2:B10)        — сумма диапазона
=C3/B3*100          — процент выполнения
=B3-B4              — разница (прибыль = выручка − расходы)
=AVERAGE(D2:D20)    — среднее значение
=IF(C3>B3,"✓","✗")  — условие
```

При экспорте в XLSX формулы записываются как настоящие Excel-формулы — они вычислятся при открытии файла.

В превью формульные ячейки отображаются фиолетовым цветом с символом `ƒ` в углу.

## Готовые шаблоны из библиотеки

Нажмите кнопку **«📚 Шаблоны»** в верхней панели.

### Финансовый отчёт
Один лист «Финансы». Включает:
- Шапку с датой и периодом
- Таблицу «Показатель / План / Факт / % выполнения» с маркерами VAL_01…VAL_04
- Строку «Прибыль» с формулами `=B3-B4`, `=C3-C4`, `=C5/B5*100`
- Маркер-таблицу TBL_01 для детализации (5 строк)
- Подвал с подписями

### Ежемесячный KPI
Один лист «KPI». Включает:
- Шапку с зелёным фоном, маркерами месяца (VAL_01) и подразделения (VAL_02)
- KPI-сетку: Выручка, Кол-во клиентов, NPS, Срок обработки — с плановыми (VAL_03…) и фактическими (VAL_04…) значениями
- Подвал

### Статистическая форма
Один лист «Статистика». Включает:
- Шапку с фиолетовым фоном, строками «Организация» и «Период» (VAL_01 — VAL_02)
- Маркер-таблицу TBL_01 на 8 строк для основного датасета
- Подвал

> После выбора шаблона текущее состояние редактора **полностью заменяется** шаблоном. Сохраните свою работу заранее через **«↑ JSON»**.

## Работа с несколькими листами

В рабочей области вверху находится строка вкладок листов.

| Действие | Как выполнить |
|----------|---------------|
| Переключить лист | Кликнуть на вкладку |
| Добавить лист | Кнопка `+` после вкладок |
| Удалить лист | Кнопка `✕` рядом с именем вкладки (доступна при 2+ листах) |

Каждый лист имеет свой набор блоков и структуру. При экспорте в XLSX каждый лист становится отдельным **листом книги Excel** с тем же именем.

## Импорт и экспорт

### Экспорт в XLSX

Кнопка **«↑ XLSX»** (зелёная) в верхней панели.

Скачивается файл `visiology_template.xlsx`. Особенности:
- Каждый лист → отдельная вкладка Excel
- Объединение ячеек (colspan/rowspan) → Excel-мержи
- Формульные ячейки → настоящие Excel-формулы
- Маркеры Visiology → текстовые метки `{{VIS_...}}`
- Ширина столбцов: A = 26 символов, B/C/D = 22 символа

### Экспорт в JSON

Кнопка **«↑ JSON»** — сохраняет полное состояние редактора (все листы, блоки, стили) в файл `template.json`. Формат:

```json
{
  "version": "1.0",
  "app": "vrb",
  "sheets": [ ... ]
}
```

Используйте для **сохранения промежуточного состояния** и переноса между сессиями.

### Импорт из XLSX

Кнопка **«📂 XLSX»** — откроет диалог выбора файла.

При импорте:
- Каждая вкладка Excel → лист в редакторе
- Объединённые ячейки → корректный colspan/rowspan
- Ячейки, начинающиеся с `=` → помечаются как формульные
- Ячейки, содержащие `{{...}}` → автоматически помечаются как маркеры нужного типа

### Импорт из JSON

Кнопка **«↓ JSON»** — вставьте содержимое ранее сохранённого `template.json` в текстовое поле и нажмите «Импорт».

## Структура проекта

```
fd_visiology-report-builder/
├── visiology-report-builder.jsx   # Весь исходный код приложения
```

Всё приложение находится в одном файле `visiology-report-builder.jsx`. Компоненты:

| Компонент / функция | Назначение |
|---------------------|------------|
| `App` | Корневой компонент, глобальное состояние |
| `CellEditor` | Всплывающий редактор ячейки |
| `MarkerPanel` | Панель со списком маркеров |

| `BLOCK_TEMPLATES` | 7 встроенных блоков-шаблонов |

## Установка и запуск

### Требования

- [Node.js](https://nodejs.org) версии 18 или выше
- npm (входит в Node.js)

### Шаги

```bash
# Клонировать репозиторий
git clone <url>
cd fd_visiology-report-builder

# Установить зависимости
npm install

# Запустить в режиме разработки
npm run dev
# → http://localhost:5173

# Собрать production-версию
npm run build
# → папка dist/

# Предпросмотр production-сборки
npm run preview
# → http://localhost:4173
```

### Переменные окружения

Приложение не требует никаких переменных окружения.

### Открытие в сети

По умолчанию dev-сервер доступен только локально. Чтобы открыть доступ по сети:

```bash
npm run dev -- --host
```
